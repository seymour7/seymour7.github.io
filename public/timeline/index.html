  • Jun-Oct 2014
    Age 22

    This project explored the design and implementation of a mobile application for South African Sign Language recognition. The application connects, via Bluetooth, to an instrumented glove in development by the University of Cape Town. The objective is to recognise the manual alphabet and manual numeric digits that have static gestures (31 signs in total). This project was split into two primary tasks: designing the classification model and designing the mobile application. Two neural networks (one with a log-sigmoid, and one with a symmetric Elliott activation function) and a Support Vector Machine (SVM) were compared. The SVM was chosen for implementation primarily because of its high accuracy of 99% and its superior robustness. However, multiple problems were encountered when attempting to automatically detect isolated gestures in a continuous stream of data. The mobile application was developed for Android and allowed the user to connect to a Bluetooth glove, display and dictate the classification output and calibrate the connected glove. It was predominately successful in real-time classification of the gestures within the restrictions of the smartphone's resources. On a low-end smartphone, the classification time did not exceed 45ms, the memory usage did not exceed 15 MB, and the battery life during typical usage was approximately 11 hours.

    Things I learnt:

    • Android app dev
    • Planning skills/UML
    • Machine learning

  • Jan-Jul 2013

    Convos pairs random UCT students, anonymously, in a chat window using UCT user/pass authentication. My most challenging project to date due to message synchronisation and the real-time requirements of the app. I developed it using PHP, MySQL and JavaScript (using JQuery).   • Mar-Jun 2008
    Age 15/16

    In 2008 I decided to make an Afrikaans dictionary so that I could instantly translate Afrikaans words into English and vice versa. However, after looking at the size of my school dictionary, I realized it would take me years to manually add each word. My next idea was to make the website community based and allow users to submit words with their translations and also allow the people that use the dictionary to remove spam/wrong words by voting the translations up or down. I made the site using PHP, XHTML, CSS, Javascript (for asynchronous rating of words/spam reporting) and MySQL for the database of all the words. I tried to keep the site as simple as possible. You can search for translations on the homepage, add translations on the translate page and browse words from a-z on the browse page. I never released the site because of hosting costs.

    Things I learnt:

    • Ajax (first with XMLHttpRequest, then jQuery)
    • Basic search
    • Database design
    • XSS prevention
    • URL rewrites with Apache
    • PHP frameworks (CodeIgniter, an MVC framework)
    • Better organisation of a website's file and directory structure

  • 2005

I can't quite recall what my motivation was, but in 2005 I began doing more constructive things (which felt like quite a natural transition). Me and my friend, Tim, began game development using a drag-and-drop application, GameMaker 5. I still remember the day Tim and I learnt what a "variable" was - it was quite eye-opening (and remember, this was before we even did algebra in high school).
